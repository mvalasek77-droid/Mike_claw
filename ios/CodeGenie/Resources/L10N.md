# Localization

CodeGenie's source language is **English**. `Localizable.xcstrings` is a Swift String Catalog (Xcode 15+) shipping with seed translations for:

- `en` (source)
- `es` Spanish
- `fr` French
- `de` German
- `ja` Japanese
- `zh-Hans` Simplified Chinese

The current catalog covers ~15 of the highest-visibility user-facing strings (splash subtitle, home hero, tab bar, primary CTAs, Pair Mac status). The other strings still live as Swift literals; that's intentional — we localize **only what users see** and grow the catalog as features ship.

## Usage in code

```swift
Text("Watch the tour")              // SwiftUI auto-resolves via the catalog
Label("Pair your Mac", systemImage: "macbook.and.iphone")
```

SwiftUI's `Text` initializer takes a `LocalizedStringKey` by default — string literals are looked up automatically. Avoid these patterns when you want localization:

- `Text(String(localized: ...))` — fine, but verbose
- `Text(verbatim: ...)` — **explicitly opts out** of localization (use for usernames, version numbers, etc.)
- Interpolated strings (`"\(x) builds run"`) — interpolation is handled by Foundation; the *format string* portion still localizes correctly.

## Adding a string

1. Use the new string in code as a plain literal (`Text("My new copy")`).
2. Open `Localizable.xcstrings` in Xcode → the catalog auto-extracts and marks every target locale as `new`.
3. Fill in translations or leave them as `new` to fall back to source.

## Adding a language

In Xcode → project settings → **Info → Localizations → +**. The catalog adds an empty column; pre-existing strings get marked `new` for the new locale.

## Apple Style Guide

Translators should follow Apple's [Style Guides](https://help.apple.com/applestyleguide/) for tone (concise, sentence case, no oxford comma) and platform-specific terminology (Settings → Réglages, not Paramètres on iOS).
